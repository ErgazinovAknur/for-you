# для тебя 💗 — милый сайт-подарок

Розовый пиксельный сайт с тремя штуками внутри:
- 🎧 **Плейлист** — можно добавлять песни онлайн (видно всем, кто заходит на сайт)
- 🗓️ **Календарь** — заметки и важные даты/дедлайны
- 🎮 **Мини-игра** — «прыг-прыг котик», аналог динозаврика из Google

## Шаг 1. Выложить на GitHub Pages

1. Создай новый репозиторий на GitHub, например `for-you`.
2. Залей туда все файлы из этой папки: `index.html`, `style.css`, `app.js`.
3. Зайди в **Settings → Pages**, в разделе Source выбери ветку `main` и папку `/root`.
4. Через минуту сайт будет доступен по адресу вида
   `https://твой-логин.github.io/for-you/`.

## Шаг 2. Подключить общий онлайн-плейлист и календарь (Firebase, бесплатно)

Без этого шага сайт всё равно будет работать, но плейлист и календарь
будут сохраняться только в браузере того, кто их добавил (через
localStorage) — то есть не будут "общими". Чтобы они стали видны
обоим и сохранялись по-настоящему онлайн, нужно 5 минут на бесплатный
Firebase:

1. Зайди на https://console.firebase.google.com и создай проект (можно
   назвать как угодно, например `nasha-lyubov`).
2. В меню слева: **Build → Firestore Database → Create database**.
   Выбери режим **test mode** (для личного некоммерческого сайта этого
   достаточно) и любой регион.
3. Слева вверху рядом с названием проекта нажми на шестерёнку →
   **Project settings**, прокрути до **"Your apps"**, нажми на иконку
   `</>` (Web), придумай имя приложения, зарегистрируй.
4. Появится объект `firebaseConfig` — скопируй его.
5. Открой `app.js` в самом начале файла и вставь свои значения вместо
   `REPLACE_ME` в объекте `firebaseConfig`.
6. Сохрани, залей изменения на GitHub — готово, плейлист и календарь
   теперь общие и сохраняются онлайн.

> Совет по безопасности: в тестовом режиме Firestore любой, кто узнает
> адрес твоей базы, теоретически может её редактировать. Для личного
> подарка это обычно некритично, но если хочешь подстраховаться —
> в Firestore Rules можно ограничить запись, например, простым паролем
> через Cloud Functions, либо просто не публиковать конфиг публично.

## Шаг 3. Немного персонализировать

- В `index.html` замени заголовок `ДЛЯ ТЕБЯ` и подпись под ним на
  что-то своё/имя.
- В `app.js` в массиве `dailyMessages` замени фразы на свои личные.
- Можно поменять цвета в `style.css` в блоке `:root` (переменные
  `--pink-*`, `--gold`, `--mint`).

## Локальная проверка перед публикацией

Просто открой `index.html` в браузере — всё заработает и без сервера
(плейлист/календарь будут в офлайн-режиме, пока не подключишь
Firebase).

Приятного подарка! 🎀
