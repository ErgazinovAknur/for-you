# для тебя 💗 — милый сайт-подарок

## Файлы

```
for-you-site/
├── index.html
├── style.css
├── app.js
└── README.md
```

Просто 4 файла, без папок — заливай все разом в корень репозитория.

## Что внутри

- 🎧 **Плейлист** — вставляешь ссылку (YouTube / Spotify / прямой .mp3), название подтягивается само. Можно указать необязательное поле "сборник" (например, имя исполнителя или настроение) — песни с одинаковым сборником сворачиваются в отдельную раскрывающуюся группу, а не висят одним огромным списком. Внизу — полноценный плеер: play/pause, вперёд/назад, стоп, повтор одного трека 🔂, повтор всего плейлиста 🔁, скорость, громкость, перемотка по клику на шкале.
- 🗓️ **Календарь** — клик по названию месяца открывает быстрый выбор месяца и года (не нужно много раз жать "вперёд"/"назад"). Текст заметки виден прямо в клетке дня, важные даты/дедлайны подсвечиваются красным. У каждой заметки в окне дня есть кнопка ✕ для удаления.
- 🏠 **Главная** — приветствие и случайная тёплая фраза дня.

## Шаг 1. Выложить на GitHub Pages

1. Создай репозиторий на GitHub (Public).
2. Add file → Upload files → перетащи все 4 файла в корень репозитория.
3. Settings → Pages → Source: `Deploy from a branch`, ветка `main`, папка `/ (root)` → Save.
4. Через 1–2 минуты сайт будет по ссылке `https://твой-логин.github.io/репозиторий/`.

## Шаг 2. Подключить общий онлайн-плейлист и календарь (Firebase, бесплатно)

Без этого шага плейлист и календарь сохраняются только в браузере
того, кто их добавил (localStorage) и не видны другому человеку.

1. https://console.firebase.google.com → создать проект.
2. Build → Firestore Database → Create database → test mode.
3. Шестерёнка → Project settings → низ страницы "Your apps" → `</>` (Web) → зарегистрировать приложение.
4. Скопируй появившийся объект `firebaseConfig`.
5. Открой `app.js`, в самом начале файла вставь свои значения вместо `REPLACE_ME`.
6. Сохрани и залей `app.js` на GitHub — плейлист и календарь станут общими и сохранятся онлайн.

> Тестовый режим Firestore означает, что теоретически любой, кто узнает адрес твоей базы, может её отредактировать. Для личного подарка это обычно не проблема, но конфиг лучше не выкладывать в публичные соцсети.

## Про сборники в плейлисте

Поле "сборник" при добавлении песни — просто текст, который ты придумываешь сам (например `Романтика`, `Для тренировки`, `Queen`). Все песни с одинаковым текстом (без учёта пустых пробелов) попадают в одну группу. Поле можно оставить пустым — тогда песня попадёт в группу "Без сборника". После добавления песни поле сборника не очищается — удобно добавлять несколько песен подряд в один сборник, меняя только ссылку.

## Про прослушивание песен

В поле ссылки подходят:
- YouTube (`youtube.com/watch?v=...`, `youtu.be/...`)
- Spotify (`open.spotify.com/track/...`) — у Spotify нет публичного API для управления воспроизведением кнопками сайта, поэтому такие треки открываются в собственном плеере Spotify по клику, без кнопок вперёд/назад/скорость.
- Прямая ссылка на аудиофайл (`.mp3`, `.ogg`, `.wav`, `.m4a`)

Треки YouTube и аудиофайлы полностью управляются общим плеером сайта. Кнопки "вперёд"/"назад" переключают между всеми песнями по порядку добавления, независимо от группы.

## Персонализация

- Заголовок и текст — в `index.html` (ищи `ДЛЯ ТЕБЯ`, тексты разделов).
- Фразы дня на главной — массив `dailyMessages` в начале `app.js`.
- Цвета — блок `:root { --pink-*; --gold; --mint; }` в самом верху `style.css`.

## Локальная проверка

Просто открой `index.html` в браузере — сайт полностью рабочий и без сервера (кроме общего онлайн-плейлиста/календаря, пока не подключишь Firebase).
