# для тебя 💗 — милый сайт-подарок

## Структура файлов (5 файлов, без папок)

```
for-you-site/
├── index.html
├── style.css
├── app.js
├── assets.js     ← картинки для игры зашиты сюда напрямую (base64)
└── README.md
```

Никакой папки `images/` больше нет — специально, чтобы ничего не
потерялось и не перепуталось при загрузке на GitHub. Все 4 картинки
(девушка, парень, фон уровня, фон заставки) хранятся прямо внутри
`assets.js` в виде текста. Просто убедись, что залил **все 4 файла
(html/css/js/assets.js)** в корень репозитория.

## Что внутри

- 🎧 **Плейлист** — вставляешь ссылку (YouTube / Spotify / прямой .mp3) — название подтягивается само. Внизу — полноценный плеер: play/pause, вперёд/назад, стоп, повтор одного трека 🔂, повтор всего плейлиста 🔁, скорость 🐢/🐇, громкость 🔉/🔊, шкала прогресса с перемоткой по клику.
- 🗓️ **Календарь** — клик по дню открывает окно для заметки; текст заметки сразу виден в самой клетке дня, важные даты/дедлайны подсвечиваются красным.
- 🎮 **Игра** — бесконечный забег: играешь за девушку (спрайт из твоей картинки), парень бежит следом и с небольшой задержкой повторяет твои прыжки. Фон уровня — из присланной тобой локации, крутится по кругу. На старте — экран с вашим совместным фото и кнопками "начать игру" / рекорд, надписи не перекрывают лица.

## Шаг 1. Выложить на GitHub Pages

1. Создай репозиторий на GitHub (Public).
2. Add file → Upload files → перетащи все 4 файла (`index.html`, `style.css`, `app.js`, `assets.js`) прямо в корень репозитория (без вложенных папок).
3. Settings → Pages → Source: `Deploy from a branch`, ветка `main`, папка `/ (root)` → Save.
4. Через 1–2 минуты сайт будет по ссылке `https://твой-логин.github.io/репозиторий/`.

## Шаг 2. Подключить общий онлайн-плейлист и календарь (Firebase, бесплатно)

Без этого шага плейлист и календарь сохраняются только в браузере
того, кто их добавил (localStorage) и не видны другому человеку.
Чтобы сделать их общими:

1. https://console.firebase.google.com → создать проект.
2. Build → Firestore Database → Create database → test mode.
3. Шестерёнка → Project settings → низ страницы "Your apps" → `</>` (Web) → зарегистрировать приложение.
4. Скопируй появившийся объект `firebaseConfig`.
5. Открой `app.js`, в самом начале файла вставь свои значения вместо `REPLACE_ME`.
6. Сохрани и залей `app.js` на GitHub — плейлист и календарь станут общими и сохранятся онлайн.

> Тестовый режим Firestore означает, что теоретически любой, кто узнает адрес твоей базы, может её отредактировать. Для личного подарка это обычно не проблема, но конфиг лучше не выкладывать в публичные соцсети.

## Про прослушивание песен

В поле ссылки подходят:
- YouTube (`youtube.com/watch?v=...`, `youtu.be/...`)
- Spotify (`open.spotify.com/track/...`) — у Spotify нет публичного API для управления воспроизведением кнопками сайта, поэтому такие треки открываются в собственном плеере Spotify по клику, без кнопок вперёд/назад/скорость.
- Прямая ссылка на аудиофайл (`.mp3`, `.ogg`, `.wav`, `.m4a`)

Треки YouTube и аудиофайлы полностью управляются общим плеером сайта.

## Про игру

- Управление: пробел или тап по экрану — прыжок через провалы в земле.
- Девушка — игрок, парень — идёт следом и повторяет прыжки с задержкой.
- Собирай розы 🌹 в воздухе — бонусные очки.
- Рекорд сохраняется в браузере (localStorage), у каждого человека свой, если Firebase не подключен.
- Хочешь заменить картинки — открой `assets.js`, замени содержимое между кавычками у нужной константы (`ASSET_GIRL`, `ASSET_BOY`, `ASSET_LEVEL_BG`, `ASSET_TITLE_BG`) на свою base64-строку. Проще всего сконвертировать картинку в base64 на сайте вроде base64-image.de и вставить результат целиком (вместе с `data:image/png;base64,...`).

## Персонализация

- Заголовок и текст — в `index.html` (ищи `ДЛЯ ТЕБЯ`, тексты разделов).
- Фразы дня на главной — массив `dailyMessages` в начале `app.js`.
- Цвета — блок `:root { --pink-*; --gold; --mint; }` в самом верху `style.css`.

## Локальная проверка

Просто открой `index.html` в браузере — сайт полностью рабочий и без сервера (кроме общего онлайн-плейлиста/календаря, пока не подключишь Firebase).
